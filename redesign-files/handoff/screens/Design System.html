The system behind the screens.

One tokens file, twelve screens, and the rules that hold them together. Editorial calm as a design language: warm cream paper, navy ink, gold accent used sparingly. Open the linked spec for the full handoff; this page is the visual reference.

North star
Chat is the home, not a feature.
The default surface is a conversation. Lists, forms, and tables are things the assistant opens — not destinations the user navigates to.
Safety rail
Every AI write is staged.
Writes go through practice_assistant_actions in pending state. Approval flips it. Never auto-execute.
Aesthetic rule
One accent. Used sparingly.
Gold is punctuation. If three things on a screen are gold, two of them are wrong.

Space & radii

1280 container · 56 gutter
5 radii steps

Sharp 2px on buttons + chips is intentional — it's what distinguishes us from the typical pill-shaped SaaS. Don't soften it. Cards step up to 8px. Pills are the only place where fully-rounded is correct.

Spacing rhythm

Use 4 / 8 / 12 / 14 / 18 / 22 / 28 / 32 / 40 / 56. No formal scale token — pick the value closest to need. Cards usually take 18–22px internal padding; sections separate with 22–36px.

Elevation

3 tokens · inset highlight + drop

Every shadow has a 1px white inset on top — it suggests paper raised off the page. Never invent new shadows. If a surface needs more depth, layer a colored glow on top of `--shadow-2`.

Refactor brief

for Claude Code

The Preact codebase under blawby-ai-chatbot/src should be reorganized around the design system. Three goals:

1 · Make tokens.css the source of truth.
Copy tokens.css into src/design-system/tokens.css and import it at the app root. Audit every Tailwind bg-*, text-*, rounded-*, and shadow-* utility — replace each with the CSS variable.
2 · Build the component library.
Create src/design-system/components/ with typed Preact components for every primitive in section 6–10 above. Build src/design-system/patterns/ for the AI moments, staged actions, citations, journey, etc.
3 · Migrate one feature at a time.
Start with src/features/practice-assistant/ — the chat home. Then matters, intakes, invoices, settings. Public intake and client portal last (they have their own visual variants).

18 · Reference

Don'ts

read these out loud

The aesthetic only holds if these stay true. When in doubt, the rules win over the spec.

!
Don't auto-execute AI actions.
Every write goes through practice_assistant_actions in pending state. The "Approve" button is mandatory and IOLTA-relevant.
!
Don't introduce a third typeface.
Source Serif 4 + Geist + Geist Mono. Any new typeface is a code smell, including system fonts.
!
Don't soften the 2px button radius.
Sharp corners on buttons + chips + inputs is the most-noticed brand decision in the system. Cards step up to 8px; pills are fully rounded.
!
Don't multiply the gold.
One --accent punctuation per visible region. If you're adding a second gold thing, you're probably wrong.
!
Don't assert without sourcing.
Every AI message ends with citation pills (matters · 4 rows) or a "grounded in N sources" verifier. The AI is auditable, or it isn't.
!
Don't use emoji in product copy.
Typography and color carry emphasis. Once a Lucide icon set is introduced, emoji should disappear from the product entirely.
!
Don't make pills clickable.
Pills are read-only status. If it needs to be tappable, it's a chip. This split keeps scannability intact.
!
Don't reach for multicolor charts.
One color (the gold) plus a flat fill (paper-2). No legends, no 3D, no gradients beyond the four already in the system.
